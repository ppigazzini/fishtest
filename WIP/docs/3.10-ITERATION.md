# Iteration plan (Milestone 10 — Jinja2 idiomatic cutover, retain legacy Mako)

Date: 2026-02-07
Status: In progress

## Goals
- Produce idiomatic Jinja2 templates for Starlette in [server/fishtest/templates_jinja2](server/fishtest/templates_jinja2).
- Preserve legacy Mako in [server/fishtest/templates](server/fishtest/templates) for rebase safety.
- Keep parity tooling for legacy Mako vs the parity Jinja2 set (moved to a temporary path).
- Retire the new Mako templates and renderer that were created only for parity work.

## Requirements
- Legacy templates remain read-only and runnable for upstream rebase safety.
- Rendering stays off the event loop (threadpool boundary preserved).
- `TemplateResponse(request=..., name=..., context=...)` remains the UI return path.
- The new Jinja2 templates are not required to be line-for-line comparable with Mako.
- Parity scripts remain valid for legacy Mako vs the parity Jinja2 set.
- Avoid large refactors in `http/api.py` and `http/views.py`; only small signature/context changes that preserve parity tracking are allowed.
- ASCII-only source where practical; use HTML entities when needed.
